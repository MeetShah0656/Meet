Meet. is structured as a workspace rather than a website.

Users should feel like they are navigating software instead of webpages.

There are no traditional pages.

Everything exists as an interconnected workspace.

Users should never lose context while exploring.

# Closing Statement

The Information Architecture of Meet. is designed to eliminate the concept of isolated pages.

Everything is interconnected.

Projects reference technologies.

Technologies reference skills.

Skills reference learning.

Learning references future goals.

The result is a living knowledge graph presented through an intuitive workspace that feels like professional software rather than a traditional portfolio.
